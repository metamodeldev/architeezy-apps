# TC-11.3: CSV Format and Encoding

**System Requirement**: [SR-11.3](../../system-requirements/export.md)

## TC-11.3.1: Special characters in element names are correctly escaped in CSV output

**Functional Requirements**: [FR-11.1](../../functional-requirements.md#fr-11-data-export)

### Preconditions

- A model is loaded containing elements with names that include special characters:
  - `Component, Alpha` (contains a comma)
  - `Component "Beta"` (contains double quotes)
  - `Service\nX` (contains a newline)
- The table view is active

### Test Steps

1. Click the **Export CSV** button
   - **Expected**: A CSV file is downloaded
2. Open the downloaded CSV file in a text editor
   - **Expected**: `Component, Alpha` is enclosed in double quotes: `"Component, Alpha"`
   - **Expected**: `Component "Beta"` is enclosed in double quotes with internal quotes escaped:
     `"Component ""Beta"""`
   - **Expected**: The element with a newline in its name is enclosed in double quotes

### Post-conditions

- CSV file is correctly escaped and parseable by standard CSV tools

### Test Data

| Field         | Value                                            |
| ------------- | ------------------------------------------------ |
| Element names | Component, Alpha / Component "Beta" / Service\nX |
| Expected      | Properly escaped CSV fields                      |

## TC-11.3.2: Exported CSV file uses UTF-8 with BOM encoding

**Functional Requirements**: [FR-11.1](../../functional-requirements.md#fr-11-data-export)

### Preconditions

- **Test Architecture** is loaded; the table view is active; at least one element name contains a
  non-ASCII character (e.g. **Komponente Ä**)

### Test Steps

1. Click the **Export CSV** button
   - **Expected**: A CSV file is downloaded
2. Inspect the raw bytes of the downloaded file using a hex editor or developer tool
   - **Expected**: The file begins with the UTF-8 BOM bytes `EF BB BF`; the non-ASCII character
     **Ä** is correctly encoded in UTF-8

### Post-conditions

- CSV file uses UTF-8 with BOM; non-ASCII characters are correctly encoded

### Test Data

| Field     | Value          |
| --------- | -------------- |
| Element   | Komponente Ä   |
| Encoding  | UTF-8 with BOM |
| BOM bytes | EF BB BF       |
