# TC-11.7: Export Progress Feedback

**System Requirement**: [SR-11.7](../../system-requirements/export.md)

## TC-11.7.1: A loading indicator appears within 100ms of starting a large CSV export (>10,000 rows)

**Functional Requirements**: [FR-11.1](../../functional-requirements.md#fr-11-data-export)

### Preconditions

- A large model is loaded with more than **10,000 elements**; the table view is active; all elements
  are visible (no filter applied)

### Test Steps

1. Click the **Export CSV** button
   - **Expected**: Within 100ms of clicking, a loading indicator (spinner, progress bar, or loading
     overlay) appears; the export button may become disabled during export

### Post-conditions

- Loading indicator appears promptly; export is in progress

### Test Data

| Field          | Value               |
| -------------- | ------------------- |
| Row count      | >10,000             |
| Expected delay | ≤100ms              |
| Indicator      | Loading spinner/bar |

## TC-11.7.2: A success toast notification appears after export completes

**Functional Requirements**: [FR-11.1](../../functional-requirements.md#fr-11-data-export)

### Preconditions

- **Test Architecture** is loaded; the table view is active; the user has triggered a CSV export

### Test Steps

1. Click the **Export CSV** button and wait for the export to complete
   - **Expected**: The loading indicator (if shown) disappears; a success toast notification appears
     with a message such as "Export complete" or "CSV downloaded successfully"; the toast is visible
     for a few seconds then auto-dismisses; the export button returns to its enabled state

### Post-conditions

- Success toast is shown after export; the UI returns to its normal state

### Test Data

| Field          | Value                            |
| -------------- | -------------------------------- |
| Model          | Test Architecture                |
| Expected toast | Export complete / CSV downloaded |
| Toast behavior | Auto-dismiss after ~3s           |
