# TC-11.5: Image Export Format Support

**System Requirement**: [SR-11.5](../../system-requirements/export.md)

## TC-11.5.1: Export dropdown presents both "Export as PNG" and "Export as SVG" options

**Functional Requirements**: [FR-11.2](../../functional-requirements.md#fr-11-data-export)

### Preconditions

- **Test Architecture** is loaded; the graph view is active

### Test Steps

1. Click the **Export Image** button in the graph toolbar
   - **Expected**: A dropdown menu appears with exactly two options: **Export as PNG** and **Export
     as SVG**; no other export format options are present

### Post-conditions

- Export dropdown is visible with PNG and SVG options

### Test Data

| Field   | Value                        |
| ------- | ---------------------------- |
| Model   | Test Architecture            |
| View    | Graph                        |
| Options | Export as PNG, Export as SVG |

## TC-11.5.2: Selecting "Export as SVG" downloads a file with .svg extension

**Functional Requirements**: [FR-11.2](../../functional-requirements.md#fr-11-data-export)

### Preconditions

- **Test Architecture** is loaded; the graph view is active; the export dropdown is open

### Test Steps

1. Click **Export as SVG** from the dropdown
   - **Expected**: A file download is triggered; the downloaded file has the `.svg` file extension
     (e.g. `test-architecture.svg` or `export.svg`)
2. Open the downloaded file in a text editor or SVG viewer
   - **Expected**: The file contains valid SVG markup (`<svg>` root element); graph nodes and edges
     are represented as SVG elements

### Post-conditions

- An SVG file is downloaded; the file contains valid SVG content representing the graph

### Test Data

| Field          | Value             |
| -------------- | ----------------- |
| Model          | Test Architecture |
| Export format  | SVG               |
| File extension | .svg              |
| File content   | Valid SVG markup  |
