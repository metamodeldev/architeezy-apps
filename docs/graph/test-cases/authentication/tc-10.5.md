# TC-10.5: Sign-Out Behavior

**System Requirement**: [SR-10.5](../../system-requirements/authentication.md)

## TC-10.5.1: Clicking "Sign out" removes the token, hides user name, and shows "Sign in"

**Functional Requirements**: [FR-10.3](../../functional-requirements.md#fr-10-authentication)

### Preconditions

- The user **John Doe** is authenticated; the header shows **John Doe** and a **Sign out** button;
  **Test Architecture** is loaded

### Test Steps

1. Click the **Sign out** button in the header
   - **Expected**: The user name **John Doe** disappears from the header; the **Sign out** button
     disappears; the **Sign in** button reappears in the header
2. Inspect `localStorage` for any authentication token
   - **Expected**: No token or credential is present in `localStorage` or session storage

### Post-conditions

- User is in anonymous state; "Sign in" is visible; token is cleared

### Test Data

| Field        | Value             |
| ------------ | ----------------- |
| User         | John Doe          |
| Model        | Test Architecture |
| After action | Anonymous state   |
| Token        | Cleared           |

## TC-10.5.2: After sign-out a confirmation toast "Signed out successfully" appears

**Functional Requirements**: [FR-10.3](../../functional-requirements.md#fr-10-authentication)

### Preconditions

- The user **John Doe** is authenticated; the header shows **John Doe** and **Sign out**

### Test Steps

1. Click the **Sign out** button
   - **Expected**: A toast notification with the text "Signed out successfully" appears; the
     notification is visible for a few seconds and then auto-dismisses; the header has returned to
     the anonymous state

### Post-conditions

- Confirmation toast appears and auto-dismisses; user is in anonymous state

### Test Data

| Field          | Value                   |
| -------------- | ----------------------- |
| User           | John Doe                |
| Expected toast | Signed out successfully |
| Toast behavior | Auto-dismiss after ~3s  |
