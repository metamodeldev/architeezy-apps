# TC-10.4: Authenticated State Display

**System Requirement**: [SR-10.4](../../system-requirements/authentication.md)

## TC-10.4.1: After token receipt and profile fetch, header shows "John Doe" and "Sign out"

**Functional Requirements**: [FR-10.2](../../functional-requirements.md#fr-10-authentication)

### Preconditions

- The application is open at `/graph/`; the user has completed authentication in the popup window;
  the auth provider has sent a token back to the application; the profile API returns
  `{ "name": "John Doe" }`

### Test Steps

1. Observe the header after the authentication popup closes and the token is received
   - **Expected**: The **Sign in** button disappears; the user's display name **John Doe** appears
     in the header; a **Sign out** button or link is visible; no error notification is shown

### Post-conditions

- Header shows "John Doe" and "Sign out"; "Sign in" button is hidden

### Test Data

| Field        | Value              |
| ------------ | ------------------ |
| Token        | Present            |
| Profile name | John Doe           |
| Header shows | John Doe, Sign out |
| Sign in      | Hidden             |

## TC-10.4.2: Profile fetch failure shows placeholder name "User" and still shows "Sign out"

**Functional Requirements**: [FR-10.2](../../functional-requirements.md#fr-10-authentication)

### Preconditions

- The application has received a valid token but the profile API call returns an error (e.g. HTTP
  500 or network failure)

### Test Steps

1. Observe the header after the token is received but the profile fetch fails
   - **Expected**: The **Sign in** button disappears; the placeholder name **User** is shown in the
     header instead of a real name; the **Sign out** button or link is still visible
2. Verify the application remains functional (model can be loaded, graph renders)
   - **Expected**: The application works normally despite the profile fetch failure

### Post-conditions

- Header shows "User" and "Sign out" even when the profile cannot be fetched

### Test Data

| Field       | Value       |
| ----------- | ----------- |
| Token       | Present     |
| Profile API | Error (500) |
| Header name | User        |
| Sign out    | Visible     |
