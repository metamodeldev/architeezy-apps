# TC-10.7: Authentication Failure and Expiry

**System Requirement**: [SR-10.7](../../system-requirements/authentication.md)

## TC-10.7.1: A 401 response during an API call clears the token and shows "Sign in"

**Functional Requirements**: [FR-10.1](../../functional-requirements.md#fr-10-authentication),
[FR-10.3](../../functional-requirements.md#fr-10-authentication)

### Preconditions

- The user **John Doe** is authenticated with a token; **Test Architecture** is loaded; the next API
  call (e.g. loading a second model or refreshing data) returns HTTP 401

### Test Steps

1. Trigger an API call that returns a 401 response (e.g. select a new model while the token has
   expired server-side)
   - **Expected**: The application detects the 401 response; the token is cleared from
     `localStorage`; the header changes from showing **John Doe / Sign out** to showing the **Sign
     in** button; the graph canvas may become empty or retain the last loaded state

### Post-conditions

- Token is cleared; user is returned to anonymous state; "Sign in" is visible in the header

### Test Data

| Field        | Value                          |
| ------------ | ------------------------------ |
| User         | John Doe                       |
| API response | 401 Unauthorized               |
| After 401    | Token cleared, Sign in visible |

## TC-10.7.2: "Session expired. Please sign in again." notification appears after 401

**Functional Requirements**: [FR-10.1](../../functional-requirements.md#fr-10-authentication),
[FR-10.3](../../functional-requirements.md#fr-10-authentication)

### Preconditions

- The user **John Doe** is authenticated; an API call returns HTTP 401 (simulating session expiry)

### Test Steps

1. Trigger the 401 response from the API
   - **Expected**: A notification with the text "Session expired. Please sign in again." (or
     equivalent) appears; the notification is clearly visible; it auto-dismisses or has a close
     button
2. Observe the header state
   - **Expected**: The **Sign in** button is visible; **John Doe** and **Sign out** are no longer in
     the header

### Post-conditions

- User sees a clear session-expired message and is returned to anonymous state

### Test Data

| Field          | Value                                  |
| -------------- | -------------------------------------- |
| User           | John Doe                               |
| API response   | 401 Unauthorized                       |
| Expected toast | Session expired. Please sign in again. |
| Header state   | Sign in (anonymous)                    |
