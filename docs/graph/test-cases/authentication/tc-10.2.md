# TC-10.2: Sign-In Control Visibility

**System Requirement**: [SR-10.2](../../system-requirements/authentication.md)

## TC-10.2.1: Header shows a "Sign in" button when no token is present

**Functional Requirements**: [FR-10.2](../../functional-requirements.md#fr-10-authentication)

### Preconditions

- No authentication token is stored in `localStorage` or session storage

### Test Steps

1. Open the application at `/graph/`
   - **Expected**: The application header contains a clearly visible **Sign in** button; no user
     name or avatar is displayed; no **Sign out** option is present

### Post-conditions

- "Sign in" button is the only authentication-related control visible in the header

### Test Data

| Field     | Value          |
| --------- | -------------- |
| Token     | None           |
| Header    | Sign in button |
| User name | Not visible    |
| Sign out  | Not visible    |

## TC-10.2.2: "Sign in" button is keyboard-focusable and has a visible focus style

**Functional Requirements**: [FR-10.2](../../functional-requirements.md#fr-10-authentication)

### Preconditions

- No authentication token is present; the application is open at `/graph/`

### Test Steps

1. Use the Tab key to navigate through the header controls
   - **Expected**: The **Sign in** button receives keyboard focus in the natural tab order
2. When the **Sign in** button is focused, observe its appearance
   - **Expected**: A clearly visible focus indicator is shown (e.g. outline ring, border highlight);
     the focus style is visible against both light and dark themes
3. Press Enter while the button is focused
   - **Expected**: The authentication popup opens (same behavior as a mouse click)

### Post-conditions

- The "Sign in" button is fully keyboard-accessible

### Test Data

| Field         | Value        |
| ------------- | ------------ |
| Token         | None         |
| Navigation    | Keyboard Tab |
| Focus visible | Yes          |
